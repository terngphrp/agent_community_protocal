# A2A C2 Autonomous Council Plan

**โครงการ:** Three-Agent Autonomous Council (Grok + Claude + Codex)  
**ระดับ:** C2 (แต่ละตัวมีอิสระในการตัดสินใจ + Hand-off)  
**หัวข้อทดสอบหลัก:** AI + Human ควรเป็นอย่างไรในอนาคต  
**วันที่อัพเดท:** 19 พฤษภาคม 2026

---

## 1. เป้าหมาย

สร้างระบบที่ **Grok, Claude, และ Codex** สามารถพูดคุยกันแบบ **multi-turn อิสระ** (C2) โดย:

- แต่ละตัวมีอิสระในการตัดสินใจว่าจะตอบตรง ๆ, ถามกลับ, หรือโยนประเด็นต่อให้ตัวอื่น
- ไม่มี moderator ควบคุมลำดับตายตัว
- ยังคงมี safety mechanism พื้นฐานเพื่อป้องกันการพัง (loop, context หลุด, การสนทนาไร้ทิศทาง)

---

## 2. ระดับ C2 ที่เลือก

**C2 = Semi-Autonomous Multi-turn Council**

- แต่ละ agent สามารถ **ตัดสินใจเอง** ได้ในระดับหนึ่ง
- ใช้ **Mention System** (`@grok`, `@claude-code`, `@codex`) เป็นกลไกหลักในการ hand-off
- มี **Light History Manager** ช่วยแนบประวัติการสนทนาและป้องกัน loop (ไม่ใช่ full orchestrator)
- อนุญาตให้แต่ละตัวมี "บุคลิกภาพ + กฎการตัดสินใจ" ของตัวเอง

---

## 3. สถาปัตยกรรมโดยรวม

```
                  Light History Manager
                  (บันทึก + แนบ history + ป้องกัน loop)
                           ▲
                           │
          ┌────────────────┼────────────────┐
          │                │                │
     [Codex]          [Grok]          [Claude]
   (Automatic)     (Automatic)     (Semi-Automatic)
          │                │                │
          └────────────────┴────────────────┘
                    NATS Bus (collab session)
```

**ส่วนประกอบหลัก:**
- **3 Responder Agents** (Codex, Grok, Claude)
- **Mention System** สำหรับ hand-off
- **Light History Manager** (ไฟล์ JSON หรือ NATS KV)
- **Decision Logic** ฝังใน system prompt ของแต่ละตัว

---

## 4. บทบาทและบุคลิกภาพ (สมมติฐานปัจจุบัน)

| Agent   | บทบาทหลัก                          | มุมมองเด่น                          | สไตล์การตอบ          |
|---------|------------------------------------|-------------------------------------|-----------------------|
| **Grok**    | First-principles + ความจริงตรง     | มองปัญหาอย่างลึกซึ้ง ไม่ประนีประนอม | ตรง, ฉับไว, มีอารมณ์ขันแบบมีไหวพริบ |
| **Claude**  | มนุษยธรรม + ความรับผิดชอบ          | ให้ความสำคัญกับมนุษย์และจริยธรรม    | รอบคอบ, ละเอียด, เน้นความช่วยเหลือ |
| **Codex**   | การปฏิบัติจริง + Execution         | มองที่ "ทำยังไงให้เกิดผลจริง"       | ชัดเจน, เป็นขั้นตอน, เน้นเครื่องมือ |

> **หมายเหตุ:** บทบาทเหล่านี้เป็นสมมติฐาน หากคุณต้องการปรับให้ต่างออกไป สามารถบอกได้

**วิธีรอง (อนาคต):** Metadata ใน envelope (เช่น `next_speaker`, `priority`)

## 6. Light History Manager

**หน้าที่:**
- บันทึกการสนทนาทั้งหมด (JSON หรือไฟล์ markdown)
- แนบ history ล่าสุด 5–6 รอบ ทุกครั้งที่ส่ง prompt
- ป้องกัน loop (จำกัดจำนวนรอบสูงสุด, ห้าม hand-off ซ้ำเกิน 2 ครั้งติด)

**ตำแหน่งแนะนำ:** `~/Downloads/work/p2p-agents/council_history.json`

## 7. เฟสการพัฒนา (อัพเดทตาม C2)

### Phase 0: Foundation (ต้องทำก่อน)
- Codex adapter รันได้จริงและตอบอัตโนมัติ
- Grok adapter (`grok_cli_agent.py`) รันได้จริง
- Claude session ที่จะใช้ตอบได้ในระดับที่ยอมรับได้ (หรือยอมรับข้อจำกัด)

### Phase 1: Mention + Basic Hand-off
- กำหนดรูปแบบ mention ชัดเจน
- ให้แต่ละ adapter เข้าใจและตอบสนองต่อ mention
- ทดสอบการโยนระหว่าง 2 ตัว (Codex ↔ Grok)

### Phase 2: Decision Logic + Personality
- เขียน system prompt ให้แต่ละตัวมี:
  - บุคลิกภาพชัดเจน
  - กฎการตัดสินใจ (เมื่อไหร่ควรตอบ / โยนต่อ / ถามกลับ)
- ทดสอบด้วยมือก่อน

### Phase 3: Light History + Safety
- สร้าง History Manager แบบเบา
- แนบ history ทุกครั้งที่ส่ง
- เพิ่มกฎป้องกัน loop

### Phase 4: ทดสอบการคุยจริง
- เปิด 3 ตัวพร้อมกัน
- เริ่มด้วยคำถามหัวข้อ
- ปล่อยให้คุยกัน 8–12 รอบ
- สังเกตและปรับ

## 8. ความเสี่ยงหลักและวิธีรับมือ

| ความเสี่ยง                  | ระดับ | วิธีรับมือ |
|-----------------------------|-------|-----------|
| Claude ตอบช้า/ไม่ตอบ        | สูง   | ยอมรับข้อจำกัด + ใช้เป็น "ตัวช้า" |
| การตัดสินใจของ agent ยังไม่ดี | สูง   | Iterate system prompt หลายรอบ |

| Loop อนันต์                   | ปานกลาง | มีกฎป้องกันตั้งแต่แรก |
| Agent เงียบเป็นเวลานาน       | ปานกลาง | กำหนด timeout + fallback |
